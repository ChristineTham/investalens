# R0-P0: Project Initialisation (Package Installation)

## Status

- Completed: 2026-06-05
- Completion note: All R0-P0 setup and verification checks were run manually in Codespaces and passed, with Neon CLI auth treated as optional via documented Codespaces workaround.

## Objective

Run all package installation and CLI initialisation commands in GitHub Codespaces. This phase produces a working Node.js project with all dependencies installed. No file customisation happens here — that's R0-P1.

## Environment

- GitHub Codespaces (Ubuntu, Node.js 22, Python 3.12)
- Package manager: `pnpm`
- All commands run in terminal

## Recommended Skills

- **shadcn** — CLI init options, Base UI configuration
- **prisma-cli** — `prisma init`, `prisma generate`
- **neon-postgres** — Neon CLI, serverless driver setup

---

## PRIORITY RULE

> **The shadcn/ui CLI is the SOLE project scaffolding tool.**
>
> ```bash
> pnpm dlx shadcn@latest init --preset b0 --base base --template next
> ```
>
> This single command creates the entire project: Next.js, Tailwind CSS v4, PostCSS, TypeScript, ESLint, `components.json`, `lib/utils.ts`, `globals.css`, `layout.tsx`, and all configuration files.
>
> **Do NOT use `create-next-app`, `npm install tailwindcss`, or any other scaffolding tool.**

## Tasks

### 1. Scaffold with nextjs, tailwind, shadcn CLI

Source: https://ui.shadcn.com/docs/installation/next

```bash
npx create-next-app@latest investalens --typescript --eslint --app
rsync -a investalens/ ./
rm -rf investalens
pnpm install tailwindcss @tailwindcss/postcss postcss
pnpm dlx shadcn@latest init --preset b0 --base base --template next
```

**Expected output:**
- `package.json` (Next.js, React 19, Tailwind CSS v4, shadcn deps, `@base-ui-components/react`)
- `tsconfig.json` with `@/*` path alias
- `next.config.ts`
- `eslint.config.mjs`
- `postcss.config.mjs` (with `@tailwindcss/postcss`)
- `components.json` (with `"base": "base-ui"`)
- `app/globals.css` (Tailwind imports + default theme)
- `app/layout.tsx` (default root layout)
- `app/page.tsx` (default home page)
- `lib/utils.ts` (cn utility)
- `.gitignore`
- `node_modules/` fully installed

### 2. Initialise Prisma ORM

Source: https://www.prisma.io/docs/prisma-orm/quickstart/postgresql

```bash
pnpm add -D prisma @types/pg
pnpm add @prisma/client @prisma/adapter-pg pg dotenv
pnpm approve-builds
```

Then initialise with PostgreSQL provider:

```bash
npx prisma init --datasource-provider postgresql --output ../generated/prisma
```

This creates:
- `prisma/schema.prisma` (with `prisma-client` generator and custom output path)
- `prisma.config.ts` (Prisma configuration file)
- `.env` (with placeholder `DATABASE_URL`)

> **Note:** Prisma ORM now uses the `prisma-client` generator (not `prisma-client-js`) and requires a driver adapter at runtime. The generated `prisma.config.ts` configures the datasource URL from env.

### 3. Set up Neon connection

Source: https://neon.com/docs/get-started/with-an-agent

```bash
npx neonctl@latest init
```

If running in GitHub Codespaces and interactive Neon auth fails, use this workaround:

1. In a local browser, create the Neon project/database in the Neon Console and copy the connection string.
2. Set the connection string directly in the repo env file used by Prisma/Next.js:

```bash
echo 'DATABASE_URL="postgresql://USER:PASSWORD@HOST/DB?sslmode=require"' >> .env
echo 'DATABASE_URL="postgresql://USER:PASSWORD@HOST/DB?sslmode=require"' >> .env.local
```

3. Continue R0-P0 without blocking on `neonctl init` (it is optional for app runtime; it is mainly for CLI/MCP convenience setup).

This wizard:
- Signs you in to Neon (or signs you up)
- Creates a Neon API key
- Installs agent skills
- Configures the Neon MCP server
- Installs the Neon Local Connect extension (for VS Code)

Then install the serverless driver (source: https://neon.com/docs/guides/nextjs):

```bash
pnpm add @neondatabase/serverless
```

### 4. Install Auth.js (NextAuth v5)

Source: https://authjs.dev/getting-started/installation

```bash
pnpm add next-auth@beta
```

Generate the `AUTH_SECRET`:

```bash
npx auth secret
```

This appends `AUTH_SECRET` to your `.env.local` file.

Install the Prisma adapter:

```bash
pnpm add @auth/prisma-adapter
```

### 5. Install UI & Data Libraries

Source references:
- recharts: https://recharts.org/en-US/guide/installation
- @tanstack/react-table: https://tanstack.com/table/latest/docs/installation
- sonner: https://sonner.emilkowal.ski/getting-started
- zustand: https://github.com/pmndrs/zustand (`npm install zustand`)
- next-themes: https://github.com/pacocoursey/next-themes (`npm install next-themes`)
- react-dropzone: https://react-dropzone.js.org (`npm install --save react-dropzone`)
- papaparse: https://www.npmjs.com/package/papaparse (`npm install papaparse`)
- zod: https://zod.dev
- date-fns: https://date-fns.org
- bcryptjs: https://www.npmjs.com/package/bcryptjs

```bash
pnpm add zod recharts date-fns bcryptjs \
  @tanstack/react-table papaparse react-dropzone \
  sonner zustand next-themes
```

> **Note:** `@vercel/kv` has been deprecated (June 2024). If Redis caching is needed, install a Redis integration from Vercel Marketplace (e.g. Upstash Redis) later.

### 6. Install Dev Dependencies

Source references:
- vitest: https://vitest.dev/guide/ (`npm install -D vitest`)
- playwright: https://playwright.dev/docs/intro (`npm init playwright@latest`)
- prettier: https://prettier.io/docs/install (`npm install --save-dev --save-exact prettier`)
- prettier-plugin-tailwindcss: https://github.com/tailwindlabs/prettier-plugin-tailwindcss (`npm install -D prettier prettier-plugin-tailwindcss`)

```bash
pnpm add -D @types/bcryptjs @types/papaparse \
  vitest @testing-library/react \
  prettier prettier-plugin-tailwindcss
```

For Playwright (interactive setup — installs browsers):

```bash
pnpm create playwright
```

When prompted, choose:
- TypeScript
- Tests folder: `e2e`
- Add GitHub Actions workflow: yes
- Install browsers: yes

### 7. Set Up Python Project with uv

Source: https://docs.astral.sh/uv/guides/projects/

Initialise a uv project in the repo root (creates `pyproject.toml`, `.python-version`, `main.py`):

```bash
uv init --python 3.12
```

> Note: `uv init` will skip files that already exist (e.g. README.md, .gitignore).

Add the analytics dependencies:

```bash
uv add numpy scipy pandas
```

This:
- Updates `pyproject.toml` with the dependencies
- Creates `uv.lock` (cross-platform lockfile — commit to version control)
- Creates `.venv/` and installs packages

Remove the generated `main.py` (not needed — Python code lives in `api/`):

```bash
rm main.py
```

### 8. Generate Prisma Client

```bash
npx prisma generate
```

> Note: `prisma generate` works without a live database — it only needs the schema file. The schema will be fully expanded in R0-P1.
