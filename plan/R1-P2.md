Validate the complete R1 MVP in GitHub Codespaces: install dependencies, run database migrations, seed test data, run all tests, and verify the application works end-to-end.

## Step 2: Database Migration

The initial `init` migration (from R0-P2) only created auth tables. The full R1 schema
needs a new migration for all models added in R1-P1a through R1-P1d.

```bash
# Generate Prisma client (picks up expanded schema)
npx prisma generate

# Apply new migration for R1 models (Portfolio, Holding, Instrument, Transaction, etc.)
npx prisma migrate dev --name r1-full-schema

# Verify all tables exist
npx prisma studio
```

> **Note:** If `prisma migrate dev` detects drift from the existing `init` migration,
> it will prompt to reset. This is safe for development — the seed script repopulates data.
