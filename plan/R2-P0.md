## Step 3: Verify Vercel Python Runtime ✅

**Important:** `vercel dev` cannot route Python functions alongside Next.js locally.
Python analytics runs as a **Vercel Service** via `experimentalServices` in `vercel.json`.

**Architecture change:** Converted from `BaseHTTPRequestHandler` to **FastAPI**:

- `api/analytics/app.py` — FastAPI app entrypoint (mounted as Vercel Service)
- `api/analytics/optimize.py` — FastAPI router (placeholder, replaced in R2-P1c)
- `api/analytics/__init__.py` — package marker

**vercel.json** uses `experimentalServices` to run Next.js + Python as separate services.
The Vercel project **Framework Preset must be set to "Services"** in the dashboard.

**Verification (deploy to preview):**

```bash
npx vercel --yes
# Use the bypass token from Vercel project settings for preview deployments
curl -s https://<preview-url>/api/analytics/health \
  -H "x-vercel-protection-bypass: <token>" | jq .
curl -s https://<preview-url>/api/analytics/optimize \
  -X POST -H "Content-Type: application/json" \
  -H "x-vercel-protection-bypass: <token>" \
```

## Key Changes from Original Plan

1. **Vercel Services** — Python functions cannot coexist with Next.js via simple `api/` directory routing. Must use `experimentalServices` in `vercel.json` and set Framework Preset to "Services" in Vercel dashboard.
2. **FastAPI** — Replaced `BaseHTTPRequestHandler` with FastAPI routers for cleaner API design.
3. **dotenv** — All scripts auto-load `.env.local` so `set -a; source .env.local; set +a` is not needed.
4. **Benchmark codes** — `^MSCI` (no Yahoo ticker) replaced with `URTH` (MSCI World ETF).
5. **Analytics data service** — Created `lib/services/analytics-data.ts` (originally planned for R2-P1a, but needed for R2-P0 verification).

## Notes for the Agent

- All `uv add` commands must run in Codespaces (not locally)
- The `api/analytics/optimize.py` placeholder will be replaced in R2-P1c
- Yahoo Finance rate limit: max 5 req/sec — respect this when fetching benchmark history
