## Task 3: Webhooks

**Schema:**
```prisma
model Webhook {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  url       String
  events    String[] // ["transaction.created", "price.alert", "dividend.received"]
  secret    String   // HMAC signing secret
  active    Boolean  @default(true)
  createdAt DateTime @default(now())
}
```

**File: `src/lib/services/webhook-dispatch.ts`**

Dispatch webhooks on events:
```typescript
async function dispatchWebhook(userId: string, event: string, payload: unknown) {
  const webhooks = await db.webhook.findMany({
    where: { userId, active: true, events: { has: event } },
  });
  for (const webhook of webhooks) {
    const signature = hmacSign(JSON.stringify(payload), webhook.secret);
    await fetch(webhook.url, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "X-InvestaLens-Signature": signature,
        "X-InvestaLens-Event": event,
      },
      body: JSON.stringify(payload),
    });
  }
}
```

**Events:**
- `transaction.created` — new transaction imported/added
- `price.alert` — watchlist price alert triggered
- `dividend.received` — dividend payment recorded
- `maturity.approaching` — bond maturing within threshold
- `portfolio.rebalance` — optimisation suggests rebalance

interface NotificationConfig {

async function sendNotification(userId: string, type: string, data: unknown) {
  const config = await getUserNotificationConfig(userId);
  if (!config[type]) return;
  
  // Email via Resend
  await resend.emails.send({
    from: "InvestaLens <alerts@investalens.dev>",
    to: user.email,
    subject: getSubject(type, data),
    html: getEmailTemplate(type, data),

}

## Notes for the Agent

- Resend free tier: 100 emails/day, 1 domain. Sufficient for alerts.
- @react-pdf/renderer runs server-side — ensure all data resolved before render
- Sharesight OAuth2: register app at api.sharesight.com/partner
- Audit log can grow large — add retention policy (delete >12 months)
- Dark mode: use CSS variables approach (works with Tailwind + shadcn/ui)
- Webhook HMAC: use SHA-256 with shared secret, include timestamp for replay protection
- Cache invalidation: clear on portfolio mutation (transaction create/update/delete)
- PDF generation is CPU-intensive — consider serverless function with higher memory
